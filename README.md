### 1. ระบบจัดการหน้าต่างแสดงผลและมุมกล้อง (Window & Camera System)
* **การจัดการหน้าต่าง (GLFW & GLAD):** ทำหน้าที่จองหน่วยความจำ สร้างหน้าต่างแอปพลิเคชัน และเตรียมความพร้อมให้กับบริบทของ OpenGL เพื่อรองรับการประมวลผลกราฟิก
* **ระบบมุมกล้อง (Camera):** คลาส `Camera` ทำหน้าที่ควบคุมมุมมอง 3 มิติ (3D Viewport) โดยรับค่าพิกัดการขยับเมาส์เพื่อปรับมุมก้มเงยและหันซ้ายขวา รวมถึงรับค่าคำสั่งจากแป้นพิมพ์ (W, A, S, D) เพื่อขยับตำแหน่งของกล้อง

### 2. ระบบแอนิเมชันโครงกระดูก (Skeletal Animation System)
ส่วนนี้เป็นกลไกหลักในการประมวลผลการเคลื่อนไหวของตัวละคร โดยอาศัยการทำงานร่วมกันของ 3 คลาสหลัก:
* **`Model`:** ทำหน้าที่แยกวิเคราะห์ (Parse) ไฟล์ 3 มิติ เพื่อสกัดข้อมูลรูปทรงเรขาคณิต (Mesh), ข้อมูลพื้นผิว (Texture), และโครงสร้างกระดูก รวมถึงค่าน้ำหนัก (Bone Weights/IDs) ที่กระดูกแต่ละชิ้นมีอิทธิพลต่อจุดยอด (Vertices) 
* **`Animation`:** ทำหน้าที่โหลดและจัดเก็บข้อมูลลำดับเวลา (Keyframes) ของการเคลื่อนไหว เช่น การเดิน หรือการเต้น
* **`Animator`:** ทำหน้าที่เป็นตัวควบคุมเวลา (Time Controller) โดยจะประมวลผลค่า `deltaTime` เพื่อคำนวณหาการแปลงเชิงพื้นที่ (Transformation) ของกระดูกแต่ละชิ้นด้วยการประมาณค่า (Interpolation) ให้สอดคล้องกับเวลาปัจจุบันของแอนิเมชัน

### 3. ระบบควบคุมสถานะตัวละคร (Finite State Machine)
ชุดคำสั่งใช้กลไก State Machine ผ่านฟังก์ชัน `updateAnimationState()` เพื่อบริหารจัดการการกระทำของตัวละคร (ได้แก่ IDLE, WALK, JUMP, DANCE):
* ตรรกะการประเมินสถานะจะขึ้นอยู่กับอินพุตของผู้ใช้งาน หากมีการกดปุ่มทิศทาง ระบบจะปรับสถานะเป็น `WALK` และส่งสัญญาณให้ `animator` ดึงข้อมูลแอนิเมชันการเดินมาใช้ หากไร้การตอบสนอง ระบบจะคืนสถานะสู่ `IDLE`
* มีการใช้ตัวแปร `previousState` ตรวจสอบเทียบกับ `currentState` ซึ่งเป็นหลักการทางวิศวกรรมซอฟต์แวร์ เพื่อป้องกันการส่งคำสั่งโหลดแอนิเมชันซ้ำซ้อนในทุกๆ เฟรมหากสถานะยังคงเดิม

### 4. ระบบฟิสิกส์และการเคลื่อนที่ (Movement & Physics Logic)
ตรรกะส่วนนี้ทำงานแยกขาดจากการประมวลผลแอนิเมชัน โดยรับผิดชอบพิกัดตำแหน่งสัมบูรณ์ของตัวละครในโลก 3 มิติ:
* **การเคลื่อนที่แนวระนาบ (แกน X, Z):** เมื่อมีการรับคำสั่งทิศทาง เวกเตอร์ทิศทางจะถูกนำไปคูณกับความเร็ว (`characterSpeed`) และระยะเวลา (`deltaTime`) ก่อนนำไปปรับแก้ตัวแปรพิกัด (`characterPosition`)
* **การจำลองแรงโน้มถ่วง (แกน Y):** ฟังก์ชัน `updateCharacterPhysics()` รับผิดชอบกลศาสตร์ในแนวดิ่ง เมื่อได้รับคำสั่งกระโดด ตัวละครจะได้รับค่าความเร็วเชิงบวกในแกน Y และในทุกๆ วงรอบการทำงาน (Frame) ระบบจะนำค่าแรงโน้มถ่วง (`GRAVITY`) มาหักล้างจนกว่าตัวละครจะกลับสู่ระดับพื้นดินที่กำหนด

### 5. กระบวนการวาดภาพและแสดงผล (Rendering Pipeline & Shaders)
นี่คือขั้นตอนสุดท้ายในลูปการทำงานหลัก (`while` loop) ซึ่งจะส่งต่อข้อมูลไปยังหน่วยประมวลผลกราฟิก (GPU):
* โปรแกรมจะนำพิกัดเชิงพื้นที่ของตัวละครมาสร้างเป็น Transformation Matrix (`model = glm::translate(...)`)
* คำสั่ง `animator->GetFinalBoneMatrices()` จะส่งออกอาร์เรย์เมทริกซ์การแปลงสภาพของกระดูกทั้งโครงสร้างไปยังหน่วยความจำของ **Vertex Shader** (`anim_model.vs`)
* GPU จะทำหน้าที่คำนวณพิกัดของจุดยอด (Vertices) ตามการดึงของกระดูก และใช้ **Fragment Shader** (`anim_model.fs`) เพื่อกำหนดเม็ดสีและลวดลายบนรูปทรง ก่อนที่จะส่งผลลัพธ์สุดท้ายออกสู่หน้าจอแสดงผล
